02 / Products

Two libraries.
One ecosystem.

The same team behind Condi's inference stack ships two open-source libraries. They share a common tokenizer, checkpoint format, and tracing API — so you can train with condi-llm and distill directly into condi-slm without rewriting a line.

Lightweight / Edge

condi-slm

v2.4.1 · 18 MB wheel

A small-language-model runtime engineered for sub-15ms first-token latency on commodity CPUs. Quantized weights, ONNX export, and a 6MB Python wheel that installs in under three seconds.

quickstart.py
from condi_slm import SLM model = SLM.load("condi-mini-1.2b") reply = model.generate( "Summarize the quarterly report", max_tokens=128, temperature=0.3 ) # first token in 9.2ms on M2

Heavyweight / Cloud

condi-llm

v0.9.0 · distributed training

A distributed training and serving framework for models up to 405B parameters. Built-in FSDP, pipeline parallelism, speculative decoding, and an RLHF loop that runs on eight GPUs instead of eighty.

train.py
import condi_llm as cllm model = cllm.AutoModel.from_pretrained("condi-70b") trainer = cllm.Trainer( model, strategy="fsdp_shard", grad_accum=32, ) trainer.fit(dataset, epochs=3)

03 / Workflow

From a thousand
GPUs to a single
laptop.

The same model can be trained at scale with condi-llm, quantized with a single function call, then deployed with condi-slm on a Raspberry Pi. Same weights, same tokenizer, same API surface.

distill.py
# Train a 70B model, distill into a 1.2B student. from condi_llm import Trainer, Distiller from condi_slm import SLM teacher = Trainer.load("condi-70b") student = SLM.empty(params="1.2b") distiller = Distiller( teacher=teacher, student=student, temperature=2.0, loss="forward_kl", ) distiller.fit(dataset, epochs=4) student.save("condi-mini-1.2b")

04 / Capabilities

What you get
out of the box.

/01

Unified tokenizer

Both libraries ship with the condi-tokenizer — a 128k-vocabulary BPE that handles 47 languages, code, and structured outputs without re-encoding. Same vocab from 1.2B to 405B.
→
/02

Speculative decoding

A small draft model proposes tokens that the verifier accepts in a single forward pass. Up to 3.4× throughput on chat workloads, with no quality regression.
→
/03

ONNX & CoreML export

One call to model.export() ships your SLM to iOS, browser WASM, or any ONNX runtime. No PyTorch dependency at inference time.
→
/04

Distributed tracing

OpenTelemetry-native. Every forward pass, KV-cache hit, and quantization error is a span. Pipe directly into Honeycomb, Tempo, or your local Jaeger.
→
/05

RLHF & DPO loops

Built-in preference datasets, reward modeling, and direct preference optimization. Run a full alignment pass on a 70B model in 22 hours using eight H100s.
→
/06

Zero-config KV cache

Paged attention, prefix caching, and disk-backed offload — all on by default. 14× longer context windows on the same hardware, with no code changes.
→
